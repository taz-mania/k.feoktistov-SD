### 1.2. Нефункциональные требования (NFR)

#### 1.2.1. Производительность и Масштабируемость

| Параметр | Требование | Обоснование |
|---|---|---|
| **RPS (Read)** | 50,000 RPS пиковая нагрузка, 10,000 RPS нормальный трафик | Популярные SKU (локомотивы) составляют до 80% трафика |
| **RPS (Write)** | 5,000 events/sec потоковых обновлений | Пиковые часы продаж, интеграция с сетью магазинов |
| **Latency (p99)** | < 50 ms для запроса по SKU+Store (cache hit) | UX мобильного приложения, конкуренция |
| **Latency (p99)** | < 200 ms для геопоиска (радиус 5км, до 20 магазинов) | Сохранение конверсии при поиске |
| **Latency (p95)** | < 5 sec latency события от POS до API | Критично для предотвращения двойных продаж |

#### 1.2.2. Доступность (Availability)

| Параметр | Требование |
|---|---|
| **SLA** | 99.95% uptime (19 мин downtime/месяц допустимо) |
| **Graceful Degradation** | При недоступности WMS — чтение из кэша с маркером `stale_data: true` |
| **Failover** | Active-Active конфигурация для read replicas, RTO < 30 sec |

#### 1.2.3. Консистентность (CAP Theorem Trade-offs)

| Аспект | Решение | Обоснование |
|---|---|---|
| **Чтение наличия** | **AP** (Availability + Partition tolerance) | Предпочтение устаревших данных (false positive) над отказом в обслуживании |
| **Резервирование** | **CP** (Consistency + Partition tolerance) | Строгая консистентность при списании остатков для предотвращения оверселлинга |
| **Eventual Consistency** | Лаг синхронизации < 5 sec | Допустимый временной разрыв для FMCG ритейла с высокой скоростью оборачиваемости |
| **Consistency Model** | BASE (Basically Available, Soft state, Eventually consistent) | Для основных операций чтения; ACID только для транзакций резервирования |

#### 1.2.4. Наблюдаемость (Observability)

| Компонент | Требование |
|---|---|
| **Метрики** | RPS, Latency (p50/p95/p99), Cache Hit Ratio (>90%), Data Freshness Lag, Error Rate |
| **Трейсинг** | Distributed tracing (OpenTelemetry) сквозь POS → Bus → Cache → API |
| **Алертинг** | PagerDuty при p99 latency > 100ms, Cache Hit < 85%, Lag > 10 sec |
| **Логирование** | Корреляционный ID для всех операций по SKU/Store; retention 30 дней |

#### 1.2.5. Безопасность и Compliance

| Требование | Описание |
|---|---|
| **Data Privacy** | Анонимизация geo-запросов клиентов, шифрование трафика TLS 1.3 |
| **Rate Limiting** | 100 req/min per client для prevent scraping конкурентами |
