=== 2.3. Состав пользователей решения

==== 2.3.1. Внешние пользователи сервиса

. *Клиенты eDA* -- Покупатели, пользователи мобильного приложения (iOS/Android). +
Поиск товара по наличию, проверка остатков в конкретном магазине, подписка на уведомления.
. *Клиенты eDA* -- Покупатели, пользователи Веб-сайта компании + 
Аналогичный функционал через браузер.

==== 2.3.2. Внутренние пользователи

. *Работник торгового зала* -- Продавцы-кассиры, мерчендайзеры, пользователи handheld-терминалов (Android). +
Проверка фактического наличия на полке, коррекция остатков при пересорте, отметка товара "закончился".
. *Работник back-офиса* -- Операторы поддержки через веб-админку. + 
Ручная коррекция остатков при конфликтах, просмотр истории изменений.
. *Управление категориями* -- Категорийные менеджеры, пользователи BI-инструментов + 
Анализ оборачиваемости, аналитика товара на складе.

==== 2.3.3. Административные пользователи (DevOps/SRE)

. *Разработчик/DevOps* -- Разработка, управлени Grafana, Kibana, CLI. +
Полный доступ к observability, настройка приложений/компонентов сервиса.
. *Администатор БД* -- Консоли управления БД. +
Доступ к мониторингу репликаций, ручное переключение при failover.
. *Администратор безопасности* -- Работа с SIEM, Vault. + 
Аудит доступа, управление секретами API ключей внешних интеграций.
