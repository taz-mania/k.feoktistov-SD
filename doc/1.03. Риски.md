### 1.3. Риски

#### 1.3.1. Технические риски

| ID | Риск | Вероятность | Последствия | Митигация |
|---|---|---|---|---|
| **TR-1** | **Рассинхронизация кэша и реальности** (Phantom Inventory) | Высокая | Продажа несуществующего товара, отмена заказа | Circuit breaker на кэше при подозрении на stale data; алгоритм predictive availability с учетом velocity |
| **TR-2** | **Отказ шины данных** (Kafka/Event Bus) | Средняя | Потеря событий продаж, накопительный эффект ошибок | Dead Letter Queues, local buffering на POS-агентах, offline reconciliation job |
| **TR-3** | **Thundering Herd** на популярные SKU | Высокая | DDoS эффект при старте акций (промо-товары) | Request coalescing, stochastic caching, protection against cache stampede |
| **TR-4** | **Split-brain в распределенном кэше** | Низкая | Различные значения наличия в разных нодах | CRDT (Conflict-free Replicated Data Types) для счетчиков остатков или strong consistency для write-through кэша |
| **TR-5** | **Неконсистентность WMS vs POS** | Высокая | Расхождение из-за пересорта, краж, повреждений | Perishtable adjusting algorithm, физическая инвентаризация с коррекцией |
| **TR-6** | **Холодный старт кэша** (Cold Cache) | Средняя | Падение performance при сбросе кэша | Cache warming job, predictive preloading для top-1000 SKU |

#### 1.3.2. Бизнесовые риски

| ID | Риск | Влияние | Стратегия |
|---|---|---|---|
| **BR-1** | **Ложноположительное наличие** (False Positive) | Репутация, Cancel Rate | Conservative availability algorithm (available_qty = on_shelf_qty - safety_stock - in_flight_orders) |
| **BR-2** | **Ложноотрицательное наличие** (False Negative) | Потеря выручки | Асинхронная проверка через fallback к WMS при cache miss |
| **BR-3** | **Недостоверность гео-поиска** | UX, недовольство клиентов | Интеграция с планограммами (planograms) для точной привязки к торговому залу, а не складу |
| **BR-4** | **Сезонные пики** (Черная пятница, Новый год) | Деградация сервиса | Capacity planning x3 от baseline, autoscaler на уровне read replicas |
| **BR-5** | **Фрод с резервированием** (Hold abuse) | Утерянные продажи | TTL на резервах (15 мин), лимит резервов per user, интеграция с антифродом |
